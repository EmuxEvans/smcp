4 SMCP is a C-based CoAP stack which is suitable for embedded environments.
5 Features include:
6 
7  * Supports draft-ietf-core-coap-13.
8  * Fully asynchronous I/O.
9  * Supports both BSD sockets and [UIP](http://en.wikipedia.org/wiki/UIP_(micro_IP%29).
10  * Supports sending and receiving asynchronous CoAP responses.
11  * Supports observing resources and offering observable resources.
12  * Supports retransmission of confirmable transactions.
13  * `smcpctl` - a powerful command line tool for browsing and configuring CoAP nodes.
14 
15 Initial focus is on correctness of implementation. Stack usage and other
16 performance optimizations will become the focus later on.
17 
18 ## Why is it called *SMCP*? ##
19 
20 Historical reasons. Don't think about it too much.

80 ## Configurability ##
81 
82 One of the goals of SMCP is to implent a full-featured CoAP library, but
83 most embedded applications don't need all of these capabilities. Because of this,
84 SMCP is designed so that you can individually enable or disable features
85 depending on your needs (See `src/smcp/smcp-opts.h`).
86 
87 For example, SMCP has the ability to have more than once instance, but embedded
88 platforms will never need more than one. Passing around a reference to a
89 global variable that will never change is wasteful, so when compiled with
90 `SMCP_EMBEDDED` turned on, we transparently (via some preprocessor magic) ignore
91 the reference to the SMCP instance from all of the functions that take it.
92 This makes it easy to use the same codebase for both embedded and non-embedded
93 applications. There are other configuration options for doing things like
94 limiting `malloc()` usage, avoiding use of `printf()` (and variants),
95 enabling/disabling observing, etc.

97 ## SMCPCTL ##
98 
99 `smcpctl` is a command-line interface for browsing, observing, and
100 interacting with CoAP devices. It is, for the most part, self-
101 documenting: just type in `smcpctl help`. You can run individual
102 commands directly from the command line when invoking `smcpctl` or you can invoke
103 with no arguments and you will enter the smcpctl shell (CLI). The
104 shell environment allows you to use familiar unix commands like
105 `ls`, `cd`, and `cat`. The CLI supports quoting and tab-completion of resource
106 names, which is incredibly handy.
107 
108 Here are a few examples of how you can use it:
109 
110 ### GET a resource ###
111 
112  $ smcpctl get coap://coap.me/large
113 
114 ### Listing the contents of a resource ###
115 
116  $ smcpctl ls coap://coap.me/.well-known/core
117 
118 ### PUT a resource and show parsed response headers ###
119 
120  $ smcpctl put -i coap://coap.me/test "Testing out smcpctl's PUT command"
121 
122 ### Observe a resource for changes ###
123 
124  $ smcpctl observe coap://vs0.inf.ethz.ch/obs
